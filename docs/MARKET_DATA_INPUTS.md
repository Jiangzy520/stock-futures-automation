# 多源数据输入说明

这份说明只介绍公开可分享的接入定位、功能分工和工程要求，不包含私有配置、真实 Token 或策略实现细节。

## 1. AllTick

### 我在系统里的定位

主实时数据源，优先承担实时推送、主扫描和自动化链路输入。

### 主要功能

- 提供实时行情接入
- 支持 WebSocket 和 REST
- 支持历史 K 线
- 更适合作为主实时源来跑策略、回测数据整理和模拟盘自动化

### 接入要求

- 需要注册账号并获取 API Key / Token
- 官方 FAQ 当前写明注册后有一周免费试用
- 官网当前也展示免费套餐，但免费额度和可用产品范围有限
- 真正要做大范围实时扫描、逐笔订阅或者更高频调用时，需要看套餐限制和连接限制

### 我这边实际使用感受

AllTick 更适合作为“主实时源”，因为它的接口形式和自动化接入方式更清晰，比较适合直接进入网站、策略输出和模拟盘执行链路。

## 2. 东方财富

### 我在系统里的定位

补充行情源和交叉验证源。

### 主要功能

- 提供网页公开行情接口可抓取的数据
- 适合做分时、K 线、名称补全、行情补充
- 可以用来和主源做交叉比对，增强网页展示和信号确认

### 接入方式

不是接官方交易柜台，而是基于东方财富网页公开行情接口做采集和封装。常见接口域名包括：

- `push2.eastmoney.com`
- `push2his.eastmoney.com`
- `datacenter-web.eastmoney.com`

### 接入要求

- 如果是网页公开接口方式，通常不需要单独申请账号
- 但它本质上不是面向开发者正式签约的交易接口，接口字段、限流、可用性都需要自己兜底
- 更适合做补充源、研究源、校验源，不适合在公开说明里写成“官方稳定交易接口”

### 如果走东方财富官方量化产品

- 可看 Choice / EMQuantAPI
- 这一路需要注册账号
- 按官方手册还涉及激活、登录以及令牌文件等流程
- 更偏正式数据产品，而不是网页抓包接口

### 我这边实际使用感受

东方财富更适合做“补充源”和“展示增强”，尤其在名称、行情补全和交叉验证上很有用。

## 3. 通达信 pytdx

### 我在系统里的定位

低成本备用扫描源。

### 主要功能

- 通过 `pytdx` 连接通达信行情服务器
- 获取 A 股行情数据
- 适合做备用扫描、备用验证和低成本冗余链路
- 在主源出现波动时，可以作为旁路校验

### 接入要求

- 需要安装 `pytdx`
- 需要自己选择可用的通达信行情服务器
- 需要自己处理连接质量、超时、重连和服务器差异
- 它更像一个开源协议客户端，不是官方提供给你签 SLA 的量化接口

### 我这边实际使用感受

`pytdx` 很适合作为备用源和验证源，成本低、灵活，但更依赖自己维护，不太适合在系统里承担唯一主源角色。

## 一句话总结

如果只用一句话概括这三路输入的分工，可以写成：

```text
AllTick 负责主实时输入，东方财富负责补充和展示增强，pytdx 负责备用扫描和冗余校验。
```
