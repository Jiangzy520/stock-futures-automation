# 股票、期货实时数据获取说明

这份公开版文档重点说明的是股票、期货实时数据获取与系统分层，不展开私有策略、内部参数或生产环境配置。

## 推荐公开表达

```text
实时数据源 -> 数据适配与标准化 -> 网站展示与状态监控 -> 模拟盘/自动化执行层
```

## 一、实时数据获取层

### 1. AllTick

定位：
- 更适合作为主实时数据源
- 更偏向标准化 API 接入和程序化调用

适用场景：
- 股票实时行情接入
- 期货及其他可交易品类的实时数据接入
- WebSocket 推送、REST 查询和历史数据补充

接入方式：
- 通过官方平台注册账号
- 获取 API Key / Token
- 根据项目规模和调用频率选择对应套餐

接入特点：
- API 形态标准，适合直接做系统主接入层
- 文档、权限、鉴权和接入方式比较清晰
- 便于直接进入网站展示、数据处理和模拟盘自动化链路

官方入口：
- 官网：https://alltick.co/zh-CN
- 文档：https://en.apis.alltick.co/

### 2. 东方财富 EMT API / 机构交易平台

定位：
- 更适合作为国内市场的一体化量化接入平台
- 更偏向终端、SDK、仿真、研究和交易协同能力

适用场景：
- 股票实时行情与历史数据获取
- 量化终端接入
- 研究、仿真、回测和交易联动

接入方式：
- 通过 EMT 官方入口提交业务申请
- 根据平台要求完成账号、终端、权限和接口开通
- 再接入对应的 SDK、终端或接口能力

接入特点：
- 更适合国内证券与期货量化终端生态
- 平台能力更完整，适合终端、仿真和交易协同场景
- 更像一套完整业务平台，而不只是单一行情 API

官方入口：
- 申请入口：https://emt.18.cn/business/business-apply-all
- 帮助文档：https://emt.18.cn/api/quant-help/guide/guide.html

### 3. 通达信 TdxQuant / tqcenter 官方公开接口

定位：
- 更适合作为国内证券行情补充源、备用源或终端型接入层
- 更偏向通达信金融终端生态和本地化行情使用场景

适用场景：
- A 股行情获取
- 财务数据和市场交易数据读取
- 期货行情和历史产品数据获取
- 本地终端协同、备用扫描和数据校验

接入方式：
- 安装支持 TQ 策略功能的通达信金融终端或专业研究版
- 通过 TdxQuant / `tqcenter` Python API 获取数据
- 基于官方文档调用行情、财务和产品数据接口

接入特点：
- 公开的是基于终端的官方 Python 接口体系
- 不是纯独立云 API，而是“终端 + 接口”的接入模式
- 更适合作为补充源、校验源和备用链路

官方入口：
- 官网：https://www.tdx.com.cn
- TdxQuant 文档：https://help.tdx.com.cn/quant/docs/markdown/mindoc-1cfsjkbf8f3is
- 数据获取说明：https://help.tdx.com.cn/quant/docs/markdown/mindoc-1cfsjkbf8f3is/mindoc-1d00kk3jsibbc.html
- 数据范围说明：https://help.tdx.com.cn/quant/docs/markdown/mindoc-1ctuhthaq5qmg/
- ETF/可转债/期货数据说明：https://help.tdx.com.cn/quant/docs/markdown/mindoc-1h13a594nhvb4/

## 二、三类实时数据接口的主要区别

### 1. 接入模式不同

AllTick：
- 更典型的云端实时数据 API 模式
- 以 API Key / Token 鉴权为主
- 更适合直接接入后端服务、网站和自动化流程

东方财富 EMT API：
- 更偏平台化、业务化接入
- 往往涉及业务申请、终端、权限和平台能力开通
- 更适合终端、仿真、研究和交易协同场景

通达信 TdxQuant / tqcenter：
- 更偏终端型接入模式
- 依赖通达信金融终端或专业研究版运行环境
- 更适合本地终端生态、补充行情和备用数据链路

### 2. 工程定位不同

AllTick：
- 更适合作为主实时数据源
- 更适合做统一接入层和主行情链路

东方财富 EMT API：
- 更适合作为国内平台化增强接入
- 更适合研究、仿真和终端协同场景

通达信 TdxQuant / tqcenter：
- 更适合作为补充源、备用源和校验源
- 更适合终端协同、本地化使用和备用扫描

### 3. 数据与生态侧重点不同

AllTick：
- 更强调标准化、多品类、程序化和低延时接入

东方财富 EMT API：
- 更强调国内量化业务链路、平台能力和终端协同

通达信 TdxQuant / tqcenter：
- 更强调通达信终端生态、A 股行情场景和本地接口使用方式

### 4. 部署与维护复杂度不同

AllTick：
- 接入成本相对更低
- 适合快速接入、快速验证和持续程序化维护

东方财富 EMT API：
- 接入流程相对更完整
- 适合有明确业务目标和平台化需求的团队

通达信 TdxQuant / tqcenter：
- 接入前提是先具备终端环境
- 更适合已有通达信终端基础的团队或个人

## 三、建议的数据分层方式

### 1. 实时数据接入层

职责：
- 接入股票与期货实时行情
- 对接不同来源的数据接口
- 保持主源、补充源和备用源的分工清晰

### 2. 数据适配与标准化层

职责：
- 统一不同来源的字段结构
- 统一时间、代码、价格和成交量等核心格式
- 为上层网站、日志、模拟盘和自动化流程提供一致的数据协议

### 3. 网站展示与状态监控层

职责：
- 展示实时数据状态
- 展示数据来源运行情况
- 展示信号快照、日志和系统状态

这一层的价值不只是展示，而是为日常排障、运行监控和链路验证提供可视化入口。

### 4. 模拟盘与自动化执行层

职责：
- 消费标准化后的实时数据或信号
- 执行去重、过滤、约束和状态回写
- 形成股票与期货的模拟盘自动化闭环

公开说明时建议强调：
- 有完整链路验证
- 有模拟盘或仿真环境验证
- 有网站监控和状态回写

不建议公开：
- 具体策略参数
- 私有风控阈值
- 内部接口地址或鉴权细节
